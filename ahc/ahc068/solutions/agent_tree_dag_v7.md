# Tree DAG V7: monotonic postprocessing of the original Tree V4

## Scope

Portfolio V6 reduced Tree V4's search time and consequently retained only a
small improvement over the original. V7 instead copies `agent_tree_exact_v4`
verbatim as its search baseline:

- `SEARCH_SECONDS = 1.02`;
- the same long-strip and local-rectangle candidate set;
- the same strict distance-decreasing prefix;
- the same checkpoint conditions and five-checkpoint limit;
- the same nine roots, eight neighbor orders, and leaf tie rules; and
- the same fully-open mesh candidate.

No prefix, checkpoint, spanning-tree, or leaf-order behavior is changed. V7
adds only a second exact-tail compression and validation layer.

## Two compression candidates

For every raw adjacent-swap tail generated by the unchanged 72-plan Tree V4
selection, V7 constructs both:

1. **Original V4 compression.** Partition the original sequence into maximal
   consecutive pairwise-disjoint batches, then fuse aligned swaps inside each
   batch.
2. **Cell-dependency DAG compression.** Reschedule commuting swaps before
   fusing them, as described below.

Both compressed tails are replayed from the corresponding checkpoint board.
Only a legal tail that finishes the board participates; when both pass, the
shorter is used. If both unexpectedly fail, the raw adjacent tail is replayed
and used as the safe fallback.

Thus each unchanged V4 prefix/tree candidate receives a tail no longer than
its original V4-compressed tail. Candidate selection then uses these actual
postprocessed lengths.

## Cell-dependency DAG

Every raw operation swaps two adjacent cells. Create one event per swap. For
each cell independently, add a dependency from the previous event touching
that cell to the current event. These are exactly the ordering constraints
between noncommuting events:

- events sharing a cell retain their original relative order;
- events on disjoint cells commute and need no dependency.

Any topological order of this DAG therefore performs the same permutation as
the raw sequence. Moreover, two simultaneously ready events cannot share a
cell—the later same-cell event would depend on the earlier one—so each ready
layer is a matching.

Within one ready layer, V7 groups:

- horizontal swaps across the same column boundary on consecutive rows into
  `H r c h 2`; and
- vertical swaps across the same row boundary on consecutive columns into
  `V r c 2 w`.

A group is extended only while all newly internal boundaries are wall-free.
The resulting rectangle performs exactly the same disjoint swaps in one output
operation. Processing all events in a ready layer and releasing their
successors yields the next topological layer.

Unlike V4 compression, this can group operations separated arbitrarily far in
the raw sequence when the intervening operations are independent.

## Correctness

The unchanged Tree V4 construction exactly sorts every checkpoint: moving the
desired card to an active leaf through its unique active-tree path fixes that
leaf permanently; deleting a leaf preserves connectedness; the last card is
correct by permutation uniqueness.

The DAG preserves the original order of all swaps sharing a cell. Every swap
whose order changes is disjoint from the swapped-over events and commutes with
them. A fused rectangle equals the product of its ready, disjoint two-cell
swaps. Its bounds, parity, and internal walls are checked during replay.
Therefore every replay-valid DAG tail has the same sorted result as its raw
tail.

V7 performs two levels of validation:

- each original-compressed and DAG-compressed tail is replayed from its exact
  checkpoint board before length comparison; and
- the selected prefix-plus-tail (or mesh candidate) is replayed again from the
  original input board before output.

## Monotonicity relative to Tree V4

For every one of the same V4 candidates, the original V4 compressor is still
constructed and retained whenever the DAG result is not both valid and
shorter. Therefore that candidate's V7 length is at most its V4 length. Taking
the minimum over the unchanged candidate set preserves this inequality.

The fully-open mesh candidate is also unchanged and is selected only after
successful replay. Hence V7 is a monotonic postprocessing improvement over
Tree V4 for identical search timing behavior.

## Bounds and runtime

Raw leaf fixing remains bounded by `400*399/2 = 79,800` operations. Compression
never increases the selected raw tail, and candidates above 100,000 are
discarded, so the output bound is preserved.

For a raw tail of `K` adjacent swaps, DAG construction, topological scheduling,
and replay are `O(K + F)` plus sorting of each ready layer, where `F` is the
number of fused output operations. At `K <= 79,800`, this deterministic
postprocessing is small relative to the unchanged 1.02-second V4 search and
fits in the remaining two-second budget. Memory usage is `O(K + N^2)` in
addition to V4's existing distance tables.

Official inputs were not executed while creating V7; only formatting and
standalone optimized compilation were performed.
